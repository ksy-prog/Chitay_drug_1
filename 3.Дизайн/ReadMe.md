## Описание проекта

Демо-версия интерфейса разработана в рамках проектирования образовательной платформы, направленной на развитие читательской грамотности пользователей через интерактивные сценарии обучения и выполнения заданий.

### Основная цель

Визуализировать ключевые пользовательские сценарии, проверить логику взаимодействия с платформой и подготовить основу для дальнейшей разработки MVP.

### Задачи интерфейса

1. Обеспечивать удобную навигацию по образовательному контенту.
2. Помогать пользователю проходить задания и отслеживать прогресс.
3. Создавать интуитивно понятную и визуально комфортную среду взаимодействия.
4. Повышать вовлеченность пользователей за счет понятной структуры и игровых элементов.
5. Демонстрировать основные механики будущего продукта команде разработки и тестовым пользователям.

## Основа проектирования

При разработке интерфейса использовались следующие аналитические и проектные материалы:

1. **UX-исследование** – изучение потребностей, болей и ожиданий целевой аудитории.
2. **CJM (Customer Journey Map)** – карта пользовательского пути для выявления ключевых точек взаимодействия.
3. **JTBD (Jobs To Be Done)** – понимание задач, которые пользователь решает с помощью платформы.
4. **User Flow** – схемы последовательности экранов и логики переходов.
5. **Аналитические материалы по читательской грамотности** – учёт методик развития чтения у детей и подростков.
6. **Референсы образовательных платформ** – анализ интерфейсов аналогичных сервисов.

Все этапы подробно размещены на платформе Holst.

## UI-kit и визуальная система

В качестве основы выбран **UI-kit Taiga UI** по следующим критериям:

- современный и минималистичный внешний вид
- наличие готовой библиотеки компонентов
- удобство интеграции в дизайн-систему
- единообразие визуальных элементов

### Учтённые принципы

- простота восприятия интерфейса
- читаемость текста
- контрастность элементов
- минимализм и отсутствие визуальной перегрузки
- адаптивность интерфейса
- интуитивность взаимодействия

### Особенности для детей и подростков

- понятные визуальные акценты
- крупные кликабельные элементы
- простая навигация
- ограниченное количество отвлекающих деталей

Преимущества:

- ускорение проектирования

- быстрое внесение изменений
