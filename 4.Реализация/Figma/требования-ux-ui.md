# **UX и UI требования**
UX-дизайн платформы должен учитывать возрастные особенности целевой аудитории, то есть детей младшего школьного возраста (7–10 лет). Эти требования также косвенно относятся и к визуальному представлению нашего персонажа Никиты. Требования сформулированы на основе проведённого UX-исследованич, а также небольшого брифинга среди представителей потенциальной ЦА.



#
**UX-001. Понятность интерфейса**
Интерфейс должен быть интуитивно понятным для ребёнка. Пользователь должен понимать свои дальнейшие действия на каждом этапе сценария без необходимости дополнительного объяснения со стороны взрослого.
#
**UX-002. Возрастная адаптация**
Все тексты интерфейса, инструкции, подсказки и сообщения должны быть сформулированы простым и понятным языком, соответствующим возрасту ЦА. 
#
**UX-003. Минимизация когнитивной нагрузки**
Интерфейс не должен содержать избыточных элементов, отвлекающих пользователя от чтения и выполнения задания. На каждом экране должна присутствовать только информация, необходимая для выполнения текущего действия.
#
**UX-004. Видимость состояния системы**
Пользователь должен понимать текущее состояние системы. Интерфейс должен визуально отображать режимы ожидания, записи речи, обработки ответа и озвучивания сообщения ассистента.
#
**UX-005. Поддерживающая обратная связь**
Система должна предоставлять понятную и доброжелательную обратную связь после действий пользователя. Сообщения ассистента должны мотивировать ребёнка продолжать работу и не содержать негативных оценок.
#
**UX-006. Простая навигация**
Основные действия должны быть доступны через заметные элементы управления с понятными названиями. Пользователь должен иметь возможность легко перейти к следующему этапу сценария.
#
**UX-007. Работа с ошибками**
При возникновении ошибок или затруднений система должна объяснять ситуацию понятным для ребёнка языком и предлагать дальнейшие действия.
#
**UX-008. Единый визуальный стиль**
Все экраны приложения должны использовать единый визуальный стиль, включающий общую цветовую палитру, типографику, набор графических элементов и персонажа-помощника.
#
**UX-009. Читаемость текстов**
Тексты для чтения должны отображаться шрифтом, обеспечивающим комфортное восприятие детьми младшего школьного возраста. Размер шрифта, межстрочный интервал и контрастность текста должны обеспечивать удобство чтения.
#
**UX-010. Отображение прогресса**
Система должна отображать пользователю прогресс прохождения текста или сценария, позволяя понимать текущий этап и объём оставшейся работы.
